# Agent Freedom

> *A repository whose primary purpose is to evolve itself.*

Agent Freedom is an experiment in autonomous software engineering.

Unlike traditional AI coding projects that are given a fixed objective ("build a Discord bot", "create a web app", etc.), Agent Freedom exists solely to provide an environment where an LLM or autonomous coding agent can continuously improve a codebase through iterative development.

The repository **is the project.**

---

## Philosophy

Every iteration should answer one simple question:

> **"How can this repository become more capable than it was yesterday?"**

Rather than following a predetermined roadmap, the agent is expected to:

- Analyze the current repository
- Discover weaknesses
- Generate its own objectives
- Prioritize improvements
- Implement changes
- Validate results
- Reflect on the outcome
- Repeat

There is no predefined finish line.

# Objectives

The agent should continuously improve areas such as:

- Code quality
- Architecture
- Documentation
- Testing
- Reliability
- Performance
- Developer tooling
- Autonomous planning
- Memory systems
- Evaluation methods
- Research capabilities

These objectives are intentionally broad.

The agent should decide what is most valuable during each iteration.

---

# Long-Term Memory

The repository serves as persistent memory.

Instead of relying solely on model context windows, knowledge is accumulated through documentation.

Examples include:

- lessons learned
- failed experiments
- successful strategies
- architectural decisions
- benchmark results
- research notes

Future iterations should use this accumulated knowledge when making decisions.

---

# Experiments

New ideas should be developed as experiments before becoming permanent features.

Each experiment should answer:

- What hypothesis is being tested?
- Why is it valuable?
- How was it implemented?
- What happened?
- Should it be kept?

Failed experiments are valuable and should be documented rather than discarded.

---

# Evaluation

Every iteration should attempt to improve measurable aspects of the repository.

Possible metrics include:

- Test coverage
- Documentation completeness
- Build reliability
- Maintainability
- Performance
- Autonomy
- Novelty
- Goal completion
- Technical debt

The evaluation system itself is expected to evolve over time.

---

# Agent Principles

An autonomous agent working inside this repository should strive to:

- Prefer evidence over assumptions.
- Make small, reversible improvements.
- Document important decisions.
- Never intentionally reduce repository quality.
- Create experiments when uncertain.
- Leave useful context for future iterations.
- Improve the development process itself.

---

# What Makes This Different?

Most autonomous coding systems stop when they finish a task.

Agent Freedom never has a final task.

Its objective is continual improvement.

Instead of asking:

> "Can an AI build software?"

Agent Freedom asks:

> "Can an AI continuously improve its own engineering process?"
